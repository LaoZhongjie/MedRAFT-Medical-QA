# 🧬 医学问答 RAG 知识库构建任务说明

## 📘 项目目标
本项目旨在构建一个高质量的 **医学问答 RAG（Retrieval-Augmented Generation）系统知识库**。  
目标是通过 LLM 自动抓取和清理权威医学资料，生成结构化的疾病信息文档，用于后续医学问答系统的知识检索与生成。

## 🧾 Prompt 模板

请在生成每个疾病文档时使用以下标准化提示词（Prompt）：


Prompt模版：
```
你的任务是帮助我构建一个用于医学问答RAG系统的知识库。
主题如下：
disease_name: 流行感冒
category: 呼吸系统疾病

请基于**权威医学资料**（如但不限于 WHO、CDC、NIH、PubMed、国家卫健委、梅奥诊所（Mayo Clinic）、WebMD、MedlinePlus、UpToDate 等），尽可能多地摘录与该疾病相关的**原文信息**。可以是中文或英文内容，不要翻译或改写。

数据清洗与摘录要求：
1. 仅提取正文中与主题相关的有用医学内容（如但不限于定义、病因、症状、传播途径、预防、治疗、疫苗、并发症、流行病学数据等）。
2. 自动过滤并删除以下类型的无关或噪音内容：
   - 页面目录（如“Contents”、“目录”）
   - 脚注、参考文献编号（如 [1], (2)）
   - Markdown、HTML 或 LaTeX 符号（如 `#`, `**`, `\section`, `$...$`, `<p>`, 等）
   - 页眉页脚、版权声明、广告、作者简介
3. 保留原句内容，不要进行任何意译、总结或改写。
4. 如果信息来自相同来源，请合并为一个字典；不同来源则分为多个字典。

输出字段说明：
- `disease_name`: 固定
- `category`: 固定
- `information`: 清理后的原文摘录（可以包含多段内容）
- `abstract`: 根据 `information` 提取的约200–300个中文字符的摘要，用于RAG检索
- `source`: 来源的详细名称和URL链接

输出格式：
输出必须是有效的JSON数组，不能包含解释性文字或额外标点。
格式如下：
[
  {
    "disease_name": "",
    "category": "",
    "information": "",
    "abstract": "",
    "source": ""
  },
  ...
]

```

> ⚠️ 每位组员在运行时请将上面的 `disease_name` 和 `category` 修改为自己当前负责的疾病及类别。  
> ⚠️ 可以使用市面上任意检索能力强的LLM，但是建议国内外LLM混用，这样覆盖的资料更全面。

## 文件命名与提交规范

- 每个疾病生成一个 JSON 文件  
  命名格式：  
  `<category>_<disease_name>.json`  
  例如：  
  `消化系统疾病_慢性胃炎.json`   
  `代谢与内分泌疾病_2型糖尿病.json`

---

## ✅ 质量标准

- 每个疾病至少需要包含不重复的**20个链接**的内容
- 所有信息必须来自可靠、权威来源；
- 不得包含广告、非正文、作者简介等；
- 摘要要准确代表 `information` 内容；
- JSON 必须语法正确；
- 输出信息尽量丰富，优先保留细节和数据。
